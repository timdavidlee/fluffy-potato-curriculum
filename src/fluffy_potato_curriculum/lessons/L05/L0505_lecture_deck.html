Slide 2.3

A conditional edge alone is not an agent

  • Be precise: every conditional edge so far branched on state you set — L05's classifier label, or direct user input. That is still a workflow.
  • It is an agent only when the edge branches on the model's own decision to keep going and that edge loops.
“But the classifier was the model deciding” — it wasn't: it wrote a label into state; your function chose the edge, and nothing looped. A conditional edge alone isn't an agent — it takes both.

Slide 2.4

Loops aren't only for agents — three back-edges, one axis

  • The agent's tool loop is the only cycle you'll build (L10/L11) — but not the only reason a graph loops back. Retry / self-correction and evaluator–optimizer are loops you wire into workflows.
  • Ask the decider question for each back-edge: who decides to go around again? A check you wrote — a pass flag, an attempt counter, a score bar — is still a workflow. The back-edge is necessary, not sufficient.
Solid cyan = a loop you own today (still a workflow — you wrote the exit condition); dashed = the one L11 draws. Same edge three times, three deciders — only the third is an agent.

When to use which
You can draw the back-edge — the skill is knowing when not to. Pick the simplest shape that solves the task.

Prefer the simplest shape that solves the task

Use a workflow when…Reach for an agent when…
the task has a known shapethe steps can't be known in advance
you want predictability + low costthe model genuinely must decide its own path
you need it testable and easy to traceopen-ended exploration is the point
(most production “AI features”)(the minority — but the loud minority)

Determinism is a feature, not a limitation. Same input, same path: predictable, cheaper, lower-latency, trivially testable.

Slide 3.2

The common failure mode

  • Watch for this: reaching for an agent when a workflow would do. It costs more, is less predictable, and is harder to debug — you'd hand the model control you never needed to give away.
  • The engineering skill is choosing the simplest shape that solves the task — and that's usually a workflow.

This is the thesis of Anthropic's Building Effective Agents: don't build an agent because you can — build one only when the task's path genuinely can't be known ahead of time.
