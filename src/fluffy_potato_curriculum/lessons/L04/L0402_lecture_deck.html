One node becomes several, wired in order

from langgraph.graph import StateGraph, END

builder = StateGraph(TicketState)        # 1. typed state schema
builder.add_node("parse", parse)         # 2. add nodes
builder.add_node("draft", draft)
builder.set_entry_point("parse")         # 3. where it starts
builder.add_edge("parse", "draft")       # 4. wire edges
builder.add_edge("draft", END)
app = builder.compile()                        # 5. compile to runnable
result = await app.ainvoke({"ticket": "..."})  #    await it on input
Same recipe as L03 — but with several nodes and edges between them. New wiring, no new primitive.

Slide 2.2

Node — a typed function that returns an update

  • A node is a plain function: it reads state and returns a partial update — a dict of just the fields it changed, not the whole state. LangGraph merges it for you.
  • A node does one unit of work and hands back an update — nothing more. What's new here: several nodes now share one state object.
async def parse(state: TicketState) -> dict[str, object]:
    # Extract structured fields (a light step → Haiku)
    reply = await haiku.ainvoke(f"Extract: {state['ticket']}")
    return {"parsed": reply.content}   # only what changed
Cyan is the one field the node changed, merged back into state — a partial update, never the whole state.

Slide 2.3

State & reducer — the data that flows

  • State is a typed object (a TypedDict) threaded through every node in turn.
  • A reducer merges each update into state, per field: the default overwrites; an Annotated[list, add] field appends — your first genuinely new primitive in L04.
  • Same machinery L11 reuses for an agent's message history — met here first.

Slide 2.4

What belongs in state — and what doesn't

  • In state: data that flows between nodes — the extracted fields, intermediate drafts.
  • Not in state: the model client and configuration. Those are dependencies you wire in at build time, not data that flows.
  • You construct a ChatAnthropic client once and close over it in the node — you don't thread it through ainvoke.

Slide 2.5

Edge, entry point, END, DAG

  • Edge — a fixed transition A → B, taken every time; L05 adds the conditional kind.
  • Entry point — where execution starts (set_entry_point).
  • END — the sentinel where every path stops.
  • DAG — no back-edges; that absence makes it a workflow, not an agent.
Same motif, this time annotated — the vocabulary pinned onto the chain.

The mechanism: a node is an independent call

  • Because each node is its own model call, you can construct your own ChatAnthropic(model=...) for each — letting you mix models per node.
  • Use a cheap, fast model (Haiku 4.5) for light steps — extract, summarize — and a capable model (Sonnet 4.6) for heavy reasoning — draft, analyze.

Slide 5.3

Determinism is a feature, not a limitation

  • Same input → same path: predictable, cheaper, lower-latency, trivially testable. A tiny eval set (the L13 discipline) is cheap and honest.
  • The model inside a node is still non-deterministic — a draft's wording varies. The path is what's stable, and that's the whole lesson.
