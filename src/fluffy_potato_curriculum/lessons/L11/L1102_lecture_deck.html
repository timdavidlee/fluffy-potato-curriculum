Slide 1.2

The one-liner, named now

from langchain.agents import create_agent

agent = create_agent(model, [calculator, lookup, flaky_fetch], system_prompt=...)
  • create_agent returns a running shallow agent: the loop, the routing, the message bookkeeping, and the step cap — all supplied. The whole lesson is this one call.
  • Shallow agent, defined: one model, one tool set, one decision point that either calls a tool or finishes. That is exactly the L10 agent graph.
  • Not a deep agent (planning / memory / reflection — that’s L18). Shallow ≠ lesser; most production agents are shallow.

L11 · SHALLOW AGENTS
Slide 4.1

You will not wire this graph by hand today

  • create_agent builds exactly this graph for you. You configure model + tools + prompt and run it; you do not assemble nodes, edges, or reducers.
  • You do not need to understand StateGraph to use create_agent — that is precisely what the one-liner spares you.
  • Assembling the graph node-by-node — state schema, add_messages reducer, explicit conditional edges — is L15’s job.

The point of the diagram is only this: create_agent is not magic — it is the graph you already wired, packaged.

Slide 4.2

The step cap did not disappear

  • A common misread: “the one-liner removed the need for a step cap.” It didn’t.
  • create_agent ships the same recursion_limit you set on invoke in L10 — default 25 loop steps; raise or lower it in the run config.
  • A runaway agent still hits that cap, and hitting it is still a signal worth investigating — exactly the L10 lesson, restated.
