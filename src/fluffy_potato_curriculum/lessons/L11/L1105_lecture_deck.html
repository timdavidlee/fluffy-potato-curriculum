The step cap is still there — just a default now

Slide 3.3

Knowing where the ceiling is is the skill

up here: explicit StateGraph — L15
branching flows belong at this altitude
under-engineering — a one-liner clinging above the line
refusing to leave create_agent when the flow genuinely branches
the ceiling — is it still one loop?
down here: create_agent — the right altitude for one loop
model · tools · system prompt — nothing more needed
over-engineering — a hand-wired StateGraph crammed below the line
the L10-objective-4 mistake: a framework where 50 lines would do
  • Two opposite mistakes bracket the line — neither reflex (always hand-build, never leave the one-liner) is right. Both mis-placements are genuine failures; the line and the right altitude stay cyan.

L11 teaches the shallow side of that line: use create_agent until the control flow stops being a single loop. L15 teaches the other side.

Three shallow-agent gotchas, named

GotchaCureWhere you saw it
"It's prebuilt, it can't run away" — trusting the one-liner not to loop forever the step cap didn't vanish, it moved behind the constructor — know where it lives; hitting it is a signal, exactly as in L10 slide 2.2 — the cap is a default now
Un-debuggable magic — can't map a bad run back to agent → tools → back-edge debug it as the L10 loop it is: read the returned messages, find the turn that went wrong — the framework hid the wiring, not the behavior section 2 + slide 1.1 — every knob has an L10 twin
Wrong altitude — hand-building when one line fits, or clinging when the flow branches stay on create_agent until control flow stops being one loop — then drop to an explicit graph (L15) section 3 — the ceiling, this whole discussion
