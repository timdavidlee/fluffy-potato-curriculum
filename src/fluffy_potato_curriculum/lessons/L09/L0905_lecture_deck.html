A stdio server config (shown, not run)

  • For a local stdio server, the config names the command that launches the server as a child process — the lowest-friction transport for local development. Exact key names vary by client; the shape is what matters.
# NOT-RUNNABLE here (no `mcp` package). Shown for wire shape only. # A client config entry describing how to launch a local stdio MCP server. server_config = { "command": "uv", "args": ["run", "python", "-m", "my_org.book_meeting_server"], "transport": "stdio", }

  • Note what is absent: any tool-specific code. The config says how to start the server, never what tools it has — the client learns those at connect time (section 2).

Connect, then ask “what do you expose?”

  • Connect, then issue list-tools: the server replies with its published tool specs — the client discovers the tool list instead of shipping with it.
# NOT-RUNNABLE here (no `mcp` package). Shown for wire shape only. async with connect_stdio(server_config) as session: await session.initialize() # protocol handshake (version negotiation) listed = await session.list_tools() # discovery: "what tools do you expose?" for tool in listed.tools: print(tool.name, "->", tool.description) print(" schema:", tool.inputSchema)
name · description · inputSchema = the L08 design surface, serialized — L0903 translates it offline.

Slide 4.1

Stop the server, watch the boundary fail

Server dies mid-conversation, prompt re-issued: the client surfaces a transport error — the tool didn't fail, the boundary did. Inline tools cannot fail this way.
FailureWhat the client surfacesModel reaction (L08 error classes)
server not runningconnection refused / no handshakereports failure; needs the operator
transport disconnectmid-call errorrecoverable — retry; often resolves on restart
version mismatchhandshake rejectedunrecoverable — fix config, not retryable
missing server credentialthe tool result is an auth errorunrecoverable for the model; operator supplies the key

Slide 4.2

Recovery is an operator action

  • Restart the server, re-run the prompt — recovered. Cross-process tools add an operational dimension inline tools don't have.
  • A well-shaped error is new context exactly as in L08; what's new is where it can originate — either side of the wire.
  • The L0902 cost-ledger row “a transport to keep healthy”, made concrete: a stopped server just broke a conversation.
Live → stopped → restarted: the fix never touches the model — someone has to keep the server alive.
