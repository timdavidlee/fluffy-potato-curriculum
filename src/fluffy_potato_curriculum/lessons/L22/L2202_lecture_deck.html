The relief valve: load it just in time

  • A skill is a capability the agent loads only when it’s relevant — instead of carrying it always-on.

Slide 2.2

What a skill actually is

  • A named, described, markdown-based bundle of instructions (plus optional scripts/resources) an agent loads just in time.
  • name — a short handle. description — states when the skill applies: the trigger; the model decides relevance from this line alone.
  • body — imperative instructions written for an agent to follow, not prose for a human.
  • optional scripts/resources — deterministic steps the skill tells the agent to run rather than reason through.

Slide 3.1

Three levels of visibility

  • Progressive disclosure is the layering that makes just-in-time loading work.
  • name + description — always in context: cheap, one line each.
  • body — loads only when the model selects the skill.
  • scripts/resources — load and run only when actually invoked.

Slide 3.3

Skills are cheap, not free

  • Each skill still costs its name + description in always-on context — a pile of them adds up.
  • Overlapping or vague descriptions make the model’s selection worse — more skills can mean worse behavior, not just more cost.
  • And a skill that does load costs an extra model turn — plus the risk the model fails to load a skill it actually needed.
  • Curate. Skills are a shelf you keep tidy, not a junk drawer.

Whether a skill works is whether its description fires

  • The description is a design artifact, not a label — the skill analogue of L08’s tool naming and schema design.
  • It must be discriminating: fire on the right inputs, stay quiet otherwise — all decided from the description alone (the body isn’t loaded yet).

Slide 4.2

The failure modes of just-in-time loading

  • Too vague → the skill silently never loads when it was needed. Too broad → it loads constantly, defeating the token savings.
  • Body assumes lost context → instructions reference state the agent no longer has once the skill loads mid-run.
  • Each is a design failure in the skill, not a model failure — which is why authoring the description is the craft.
request→ description matches→ body loads 4.1’s healthy fire
request→ description too vague — no match⇢ body never arrives needed, never fired
every request→ description too broad→ body loads every turn token savings — gone
