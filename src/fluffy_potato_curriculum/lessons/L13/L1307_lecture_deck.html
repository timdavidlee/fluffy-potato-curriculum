Slide 1.2

Why a ratchet, not a one-time test

  • A unit test pins a deterministic output once. An eval set is a ratchet: once a case passes, any later change that breaks it is a regression you catch before shipping — re-run the dataset as an experiment; run comparison flags the pass→fail.

Slide 4.1 · continued

Two families — and the brittle check

  • Two families run underneath: #1 + #4 are the same discipline — eval design starts in the trace (L12's side); #2 + #3 + #5 are the “don't trust one number” family — one run, one judge, one terminal table are each a single point you shouldn't build on.
  • One cross-cutting fault sits inside the scorers — the over-tight (brittle) check: an exact-string match red-flagging a correct-but-reworded answer trains you to ignore reds, worse than no check. A single green on an imagined happy-path case is #1, #2, and #5 at once.
