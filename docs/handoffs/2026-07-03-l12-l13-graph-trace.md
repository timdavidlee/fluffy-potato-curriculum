and L13 (eval) still instrument/score `common/agent_loop.py`. Make the **graph** the traced +

### Patterns to apply

- **Imports:** `from ...common.agent_loop import run` → `from ...common.agent_graph import run`
  (same signature: `run(model, TOOLS, task, max_steps=...)`). Where a lecture builds/streams a graph,
  use `agent_graph.build_agent(...)` + `trace_graph(...)`.
- **Prose:** "the loop you built in L10" / "hand-rolled loop" / "model→tool→model loop in plain
  Python" → "the graph you built in L10". "iteration" → "`agent`-node visit" where it means a model
  call. Instrumentation is a wrapper *around* the compiled graph's stream, not emits inserted into a
  loop body.
- **`FakeModel` + `add_messages` gotcha:** any runaway/`max_steps` fixture must script **distinct**
  replies (distinct tool-call ids `r0, r1, …`); repeating one identical `AIMessage` is id-deduped and
  terminates early. See `tests/common/test_agent_graph.py::_runaway_model`.
- **Behavioral note at the cap:** with `recursion_limit = 2*max_steps`, a runaway yields `max_steps`
  `llm` spans **and** `max_steps` `tool` spans (parity with the loop) — regenerate any saved failure
  fixtures against the graph rather than reusing loop-era ones.
